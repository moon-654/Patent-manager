# 직무발명 통합 관리 시스템 DB·API 초안

## 1. 문서 목적

본 문서는 직무발명 통합 관리 시스템의 백엔드 구현을 위한 핵심 데이터 모델과 API 초안을 정의합니다. 실제 개발 시 DBMS, ORM, 인증 방식, 파일 저장소 구조에 맞춰 상세 설계가 조정될 수 있으나, 본 초안은 엔터티 경계와 서비스 책임을 정리하기 위한 기준안입니다.


## 2. 핵심 도메인

| 도메인 | 설명 |
|---|---|
| 사용자/권한 | 사용자, 역할, 권한, 데이터 접근 범위 |
| 발명 신고 | 신고서, 공동 발명자, 첨부, 승계 결정 |
| 심의/평가 | 평가표, 평가항목, 회의록 |
| 특허 관리 | 특허 마스터, 상태 이력, OA 사건, 서류 |
| 보상 관리 | 보상 건, 산정 결과, 배분 결과, 지급 이력 |
| 정책 관리 | 정책 버전, 등급 테이블, 평가 기준, 산식 |
| 규정 변경 | 공지, 전자 동의, 버전 비교 |
| 비용/유지 | 비용 항목, 연차료 스케줄 |
| 공통 | 알림, 감사로그, 파일, 내보내기 이력 |


## 3. 핵심 엔터티 관계

```text
User
 ├─< InventionSubmission
 │    ├─< SubmissionInventorShare
 │    ├─< SubmissionAttachment
 │    ├─< SuccessionDecision
 │    └─< Evaluation
 ├─< PatentMaster
 │    ├─< PatentStatusHistory
 │    ├─< OAEvent
 │    │    └─< OADocument
 │    ├─< PatentDocument
 │    ├─< CostItem
 │    └─< RewardCase
 │         ├─< RewardDistribution
 │         ├─< RewardAdjustment
 │         └─< RewardPayment
 └─< PolicyVersion
      ├─< PolicyRewardMatrix
      ├─< PolicyEvaluationCriterion
      ├─< PolicyScoreGrade
      ├─< PolicyCutoffRule
      ├─< PolicyFormulaRule
      ├─< PolicyAnnouncement
      └─< PolicyConsent
```

## 6. API 공통 규칙 제안

1. 모든 목록 API는 `page`, `size`, `sort`, `keyword`, `status` 필터를 공통 지원합니다.
2. 민감정보는 서버에서 권한 검증 후 마스킹된 필드로 응답합니다.
3. 정책/보상/권한/다운로드 관련 변경 API는 `reason` 필드를 선택 또는 필수로 받아 감사로그에 남깁니다.
4. 정책 버전과 보상 산정 API는 계산 기준 스냅샷을 응답에 포함해 프론트에서 투명하게 표시할 수 있어야 합니다.


## 7. 우선 구현 권장 범위

### 1차 MVP

1. 발명 신고/접수/승계
2. 평가표/회의록
3. 특허 마스터/OA/서류
4. 보상 건 생성/산정/배분/통지
5. 정책 버전/등급 테이블/평가 기준/산식 설정

### 2차 확장

1. 규정 변경 전자 동의
2. 비용/연차료
3. 고급 리포트/내보내기
4. 보관 정책 자동 파기/아카이빙
