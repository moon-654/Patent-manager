# 직무발명 통합 관리 시스템 MVP 설계

## 배경

기존 폴더에는 기능명세서, 화면구조서, DB/API 초안, 마인드맵만 존재했고 실행 가능한 코드가 없었다. 따라서 이번 작업의 목표는 문서 기반 그린필드 MVP를 바로 실행 가능한 웹/API 모노레포로 전환하는 것이다.

## 구현 방향

- 모노레포 구조: `apps/api`, `apps/web`
- API: NestJS, Swagger, Prisma 스키마, 역할 기반 워크플로우 엔드포인트
- 웹: Next.js App Router, 역할별 대시보드, 핵심 목록/상세/작성 화면
- 데이터 모델: 문서의 엔터티 초안을 Prisma 스키마로 고정
- 실행 전략: 로컬 데모 가능성을 위해 API는 인메모리 상태 저장을 우선 사용하고, Prisma는 영속 모델 및 후속 DB 연결 기준으로 유지

## 포함 범위

- 로그인, 역할별 홈
- 발명 신고 작성/제출/승계
- 평가표/회의록
- 특허/OA/서류
- 보상 생성/산정/승인/지급
- 정책 버전/평가기준/보상매트릭스/산식 설정
- Swagger/OpenAPI, 샘플 시드, 테스트

## 제외 범위

- SSO
- ERP/회계 연동
- 고급 리포트/내보내기
- 비용/연차료 전체 워크플로우
- 자동 파기 및 규정 전자 동의의 완전 구현

## 핵심 원칙

- 상태 전이와 계산 근거는 항상 화면에 드러나야 한다.
- 보상 계산에는 정책 버전과 스냅샷 개념이 따라붙어야 한다.
- 발명자는 본인 정보만, 관리자는 운영 전반만 볼 수 있도록 역할별 노출을 강제한다.
